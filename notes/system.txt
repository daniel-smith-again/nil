What access to I have to my underlying system?
If I have the ability to call any routine from the host API, then do I need
abstractions?
Yes, I need portable abstractions.
I especially need portable abstractions for things that NIL does innately.
Filesystem, Linking, Assembly, Memory, and Multithreading.

I have a process.
The process can spawn threads.
Threads are based on functions so they run until they return.
How do I deal with the return values?
They don't return.
What about a thread nursery?
Those also don't return.

Processes can spawn other processes.
This lets you do things like CD or use different permissions or whatever.

Filesystem:
I can open files, list directories, and pull file status (system dependent)

Linking:
I can pull in a library and I have a CFFI to call into it.

Assembly:
I can write inline assembler.

Memory:
I can claim memory and use views to store stuff into it.
