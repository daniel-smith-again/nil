Introduction

This document formally defines the Nil programming langauge.

The overall definition is separated by scope into four sections by order of dependence...

  1. Foundation
  2. Default
  3. Supplement
  4. Auxiliary

The Foundation is concerned with the form, format, and interpretation of Nil programs. A program's behavior can be completely and rigorously determined by the meanings and effects described by the Foundation.

The Default is concerned with how the facilities of Nil are exposed to the programmer.

The Supplement and Auxiliary scopes pertain to modules which may be used as "libraries" containing useful programs for constructing large or complex systems.

Foundation

The Foundation is given in parts according to each unique abstraction that Nil provides for specifying the behavior of a program. A system of types is used to connect each of these component abstractions with relations and equivalences.

The individual abstractions provided by Nil are, in no particular order, as follows...

  • Programs
  • Modules
  • Functions
  • Control
  • Effects
  • Types
  • Numbers
  • Boolean Values
  • Data
  • Sequences
  • Bytes

In addition, the Foundation also specifies the base syntax, the effects of evaluation, and the properties of correctly constructed programs.
