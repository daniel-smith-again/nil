                                ╽ Introduction │
╭──────────────────────────────────────────────╯
├ This document provides the formal definition of the algorithmic language,
│ "NIL"

┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃ A recursive acronym for "Nil Isn't Lisp". ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

, which is a recursive acronym for "Nil Isn't Lisp".
│
├ The purpose of this document is to promote the portability of NIL programs
│ across a variety of implementations and hardware.
│
│                               ╭─────────────╮
│                               ╽ Conformance │
│╭────────────────────────────────────────────╯
│├ This document in its entirety constitutes a body of requirements for an
││ implementation.
││
│├ An implementation of the NIL programming language is said to be a
││ "conforming implementation" if it fulfills the requirements as set out in
││ this document.
││
│├ Requirements which are beyond the scope of this document are explicitly
││ classified as "implementation dependent".
││
│├ Requirements which must be present in some implementation dependent capacity
││ are explicitly classified as "implementation defined".
│╰╼
╰╼

                                 ╭──────────╮
                                 ╽ Programs │
╭───────────────────────────────────────────╯
├ Programs are the medium through which the behavior and effects of execution
│ can be specified. Programs are also a form of Data.
│
├ Programs are represented by a structure composed of Symbols and Expressions.
│ Symbols are atomic and cannot be decomposed into smaller structures.
│ Expressions are composite and contain sequences of other Symbols and
│ Expressions.
│
├ A "written Program" refers to the format used for storage and transmission of
│ NIL programs. Written Programs are represented by UTF-8 encoded Unicode® 
│ character sequences.
│
├ Syntax is used to draw a correspondence between a Program and its written form.
│
│                                  ╭────────╮
│                                  │ Syntax │
│╭─────────────────────────────────┴────────╯ 
│├ The syntax of written programs is comprised of two rules...
││    
││ 1. A Symbol is any unbroken sequence of Unicode® glyphs excluding the  
││    character codes defined in the Character Category, "Z", and the ASCII round 
││    braces, "(" and ")". For berevity, the term "Blank" will be used to refer
││    to the set of characters in the Character Category, "Z".
││
││ 2. An Expression is a closed pair of round braces, "(" and ")" with an 
││    arbitrary number of characters between them.
│╰╼
│                                 ╭─────────╮
│                                 ╽ Parsing │
│╭──────────────────────────────────────────╯
│├ Parsing rules are used to reconstruct a Program from its written form. These
││ rules are as follows...
││
││ 1. A Symbol will include all glyphs until a Blank or Expression occurs.
││
││ 2. Each opening round brace corresponds to a unique Expression which will 
││    include all Symbols and Expressions occurring before the corresponding 
││    closing round brace as items in the order that they occur in the written
││    Program.
