Introduction

This document formally defines Nil with the purpose of facilitating the portability of Nil programs across a variety of implementations and environments.

Nil is a "general purpose" programming language. Programs are expressed as the applications of functions over data such that transformations are effected on that data.

A Nil program is required to conform to a notion of correctness whereby a correct program is guaranteed to never perform undesired transformations on data, or otherwise corrupt or degrade the form of the data.

Implementation

An implementation of Nil is any program which exhibits the structure and behavior set out in this document.

The Standard Definition of Nil constitutes a body of requirements that an implementation must conform to. Conversely, a conforming implementation is guaranteed to exhibit the structure and behavior described by this document.

Language Overview

The Nil programming language is comprised of modular, but interdependent, abstractions which are composed to describe a complete and correct program. These abstractions are, in no particular order...

  • Programs
  • Modules
  • Functions
  • Data
  • Types
  • Control Flow
  • Sequences
  • Value Arithmetic

Programs

Programs in Nil are both a means of specifying transformations of data, and a form of data themselves. This is such that a Nil program may represent both the transformations to be effected on data, and the data being transformed.

Written Programs

Programs as data are canonically represented as a string of characters. This format can be considered to be the authoritative representation of a program, referred to as a "written program".

Written programs are formed according to a synatx structure, which constrains the order and sequence in which certain characters may be used. Within the syntax structure are syntax elements where each element corresponds to a structural component of a program or instance of data.
