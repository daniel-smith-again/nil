                                  Introduction

This document provides the formal definition of "NIL", an algorithmic language
for computer programming.

NIL is a recursive acronym which stands for "NIL Isn't Lisp".

The purpose of this document is to promote the portability of NIL programs
across a variety of implementations and hardware.


                                     Scope

This document specifies the format, meaning, validation, and interpretation of 
NIL programs.

This document does not specify requirements for the host system, hardware, or 
the internal mechanisms of an implementation.


                                  Conformance

This document in its entirety constitutes a set of requirements for an 
implementation. 

An implementation of the NIL programming language is said to be a 
"conforming implementation" if it fulfills the requirements as set out in this
document.

Requirements which are beyond the scope of this document are explicitly
classified as "implementation dependent".


                                    Overview

NIL is both a language and a set of capabilities which together provide an 
environment which a programmer will interact with to create, evaluate, and 
reason about programs.

NIL as a language provides functions, recursive data, natural and integer numbers
of both fixed and arbitrary precision, modules, and first class continuations.

NIL as a programming environment provides an image-based, live environment which
includes an interpreter, debugger, compiler, linker, and extractor. 
These utilities are exposed as facets of the language and may be invoked 
manually by the programmer, or programmatically by the evaluation of a NIL 
program.
